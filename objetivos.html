Ingeniería de software

Es la aplicación de enfoque sistemático, disciplinado y cuantificable hacia el desarrollo, operación y mantenimiento del software.

Fase de desarrollo de software

Conjunto de principios básicos que gobiernan cada área de la tecnología e incluyen actividades de modelado y otras técnicas descriptivas.
Fases de desarrollo del software:
-Análisis y definición de requerimientos.
-Diseño del sistema.
-Desarrollo.
-Pruebas del sistema

Análisis y definición de requerimientos.

Se refiere a entender el problema y capturar los requerimientos funcionales, entidades y actividades.

Diseño de sistema.

Proceso creatico donde se determina las tecnologias de programacion y computo y se define la estructura del producto.

Calidad de software.

Es aquel que cumple con los requerimientos del usuario final, es decir el software entregado funciona como se espera que funcione.

Herramienta en la Ingeniería de software.

Instrumento o sistema automatizado para realizar de manera más exacta, eficiente y productiva una tarea durante el ciclo de vida de desarrollo de software.

Proceso en la ingeniería de software.

Es la definición de un conjunto de actividades formales que permiten transformar los requisitos de un cliente/usuario en un sistema software.
